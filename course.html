Module 2: The Data Journey

How News Turns into Data

When the Crawler finds an article, it doesn't just "send it" to the NLP service. That would be too risky—what if the NLP service is busy? Instead, they use a "Postal System."

1

Storage (S3)

The crawler saves the full HTML article in an S3 "Cloud Bucket." It's like a filing cabinet that never gets full.

2

Notification (SQS)

It then leaves a tiny "sticky note" in an SQS Queue: "Hey, there's a new file waiting in the cabinet at this location!"

3

Processing (NLP)

The NLP service checks the queue, grabs the "sticky note," and goes to the filing cabinet (S3) to read the full article.

"Wait, why not just send the article directly to the NLP service?"

Because of Reliability! If the NLP service crashes, the "sticky notes" stay in the queue. When it comes back online, it just picks up where it left off. No data lost. ✅

Module 3: Code Deep Dive

The "Decay" Formula

How do we decide if an area is "Safe"? We use a math trick called Exponential Decay. It means a robbery today is SCARY (100% weight), but a robbery from 3 months ago is less relevant (10% weight).

Module 4: The Foundation

Cloud & Infrastructure

We don't just "run" this on a laptop. We use **AWS (Amazon Web Services)** to make it live for the world. But instead of clicking buttons in a dashboard, we use Terraform.

EC2 (The Server)

The computer in the cloud where our Docker containers live and breathe.

RDS (The Brain)

Managed PostgreSQL with PostGIS. It remembers every crime and location securely.

Terraform (IaC)

"Infrastructure as Code." We write text files, and Terraform builds the cloud for us! 🛠️

Why Infrastructure as Code (IaC)?

If we ever want to move our server or rebuild it, we just run one command instead of spending hours clicking buttons. It's like having a blueprint for your entire cloud city.
